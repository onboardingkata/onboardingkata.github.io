Actualizar órdenes

Formiik Documentation : Actualizar Órdenes

Nombre del servicio Parámetros Tipo de Dato ¿Requerido? Descripción Dato Valor de retorno Descripción servicio
UpdateExistsWorkOrdersId clientId string SI Identificador con el cual será reconocido el cliente Identificador del paquete Se liberará en el: Historia El servicio permite actualizar los parámetros de una orden, en caso de que la orden exista se actualizarán/complementarán los parámetros. Suponiendo una carga inicial ParametroA = "Valor A", ParametroB = "Valor B" ParametroC = "Valor C" Datos de actualización: ParametroB = "Valor BB" ParametroD = "Valor D" Parámetros de la orden luego de la actualización: ParametroA = "Valor A", ParametroB = "Valor BB" // dato actualizado ParametroC = "Valor C" ParametroD = "Valor D" // Nuevo valor
  • no se pueden borrar parámetros.
  • Los parámetros existentes se reemplazan
  • Lo parámetros nuevos se añaden a los que ya existían.
  • Si una orden no existe o no está en una estado que pueda actualizarse. sólo se genera el log y no se inserta una nueva orden
  • Si ocurren errores al procesar un paquete, el detalle se envía al servicio de recepción de errores del cliente, siempre y cuando el cliente tenga la opción configurada.
productId string SI Identificador con el que se reconocerá la aplicación del cliente
workOrders string SI XML con N número de órdenes a actualizar.
UpdateWorkOrdersXML (Deprecado) Actualiza órdenes en formiik.
UpdateWorkOrdersXMLId clientId string SI Identificador con el cual será reconocido el cliente Identificador del paquete Este servicio sirve para actualizar los datos de las órdenes. NOTA: para la actualización sólo se actualizan los parámetros de la orden en base a su Id pero es necesario enviar toda la información ( como si fuera una orden nueva) por si se diera el caso de en que la orden no exista en un estatus en que pueda responderse, en tal escenario, se crea una nueva orden con los datos recibidos.
productId string SI Identificador con el que se reconocerá la aplicación del cliente
workOrders string SI XML con N número de órdenes a actualizar.

Restricciones:

  • Peso máximo: El parámetro WorkOrders de la operación UpdateWorkOrdersXMLId es un string que contiene un texto XML, al que no se le hace validación de tamaño directamente por parte del servicio, por tanto el peso máximo soportado se rige por la configuración del servidor en la nube que en el momento es de 2GB (2,147,483,647 bytes). Se debe tener en cuenta que también hay restricciones de peso del lado del cliente al consumir un Servicio Web (Request/Response), que usualmente están a nivel de la configuración del EndPoint y del servidor.
    La recomendación es que el contenido XML del parámetro WorkOrders no supere los 3.5MB y que máximo tenga 100 ordenes. Lo que implica que en el envío masivo de ordenes el cliente tenga que enviar las ordenes por tandas.
  • ExternalID unicó: No se garantiza la unicidad de los ExternalID de un cliente en la nube si se envían repetidos en el mismo paquete o en paquetes diferentes que se procesen en Formiik con una diferencia de pocos segundos. Por lo cual es responsabilidad del cliente garantizar que los ExternalID que envíe sean únicos.
