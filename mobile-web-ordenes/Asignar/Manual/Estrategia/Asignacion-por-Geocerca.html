Asignación por Geocerca

Formiik Documentation : Asignación por Geocerca

Dirección de la estrategia:

Esta funcionalidad se originó por: Requerimiento - Asignación por Geocerca

Ésta estrategia permite asignar órdenes a usuarios por medio de una geocerca, para ello, el usuario debe tener asignada la geocerca.

Las condiciones para que la distribución de las órdenes sea adecuadas es:

  • Las órdenes son asignadas a un grupo.
  • No existe una restricción en cuanto a la cantidad de órdenes o cantidad de usuarios que se pueden asignar con esta estrategia.
  • Para ejecutar éste algoritmo se requiere al menos una orden y al menos un usuario/geocerca.
  • Los usuarios a quienes se requiere asignar órdenes deben tener sólo una geocerca asignada.
  • Cada geocerca sólo debe tener a un usuario asignado, si la geocerca tiene a más de un usuario las órdenes se asignarán sólo a uno.
  • Las geocercas no se deben solapar entre si, y en caso de que se solapen, la orden se asignará a la primer geocerca en donde se localice.
  • Las geocercas deben ser figuras geométricas regulares, es decir; no deben tener huecos.

Éste algoritmo trabaja de la siguiente forma:

  1. Se hace un recorrido del listado de geocercas para ubicar a las órdenes dentro de una de las gocercas, para esto se usa el algoritmo: http://alienryderflex.com/polygon/
  2. Una vez ubicada la orden en la geocerca se busca en el listado de usuarios al primer usuario que tiene relacionada la geocerca.
